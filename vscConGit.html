Usar Git en VisuaL Studio Code para repositorios en gitHub

Para trabajar con nuestros repositorios de gitHub en VSC lo primero que tenemos que hacer el clonarlo en local

Para clonar un repositorio la mejor opcion es usar la "tableta de comandos" instalada en vsc y utilizar el comando Git: Clone

Toca escribir la url del repositorio para ello nos iremos a nuestro proyecto de gitHub y la copiaremos

Escribimos esta url en la "paleta de comandos" y automaticamente el programa te preguntara en que carpeta local quieres guardar los datos para trabajar con ellos

Seleccionamos la carpeta a nuestro gusto,y le damos a open en vsc,una vez hecho esto tenemos nuestro repositorio cargado de github en vsc

Ahora vamos a explicar unos comandos basicos para utilizar correctamente la terminal de vsc con git

Comandos terminal vsc con git

Git init

git init creará un nuevo repositorio local GIT. El siguiente comando de Git creará un repositorio en el directorio actual

git clone

se usa para copiar un repositorio. Si el repositorio está en un servidor remoto, la estructura es : "git clone nombredeusuario@host:/path/to/repository"

git add

se usa para agregar archivos al área de preparación.

Si se usa para agregar un solo archivo modificado,se escribira el nombre del archivo detrás ej: "git add prueba.txt"

si se usa para agregar todos los archivos modificados se escribira "git add ."

git commit

git commit creará una instantánea de los cambios y la guardará en el directorio git.

es muy aconsejable añadir detras del comando "-m + "texto" " para ponerle nombre al commit y asi localizarlo mas facilmente en caso de necesitar volver

git status

git status muestra la lista de los archivos que se han cambiado junto con los archivos que están por ser preparados o confirmados.

git push origin "master"

se usa para enviar confirmaciones locales a la rama maestra del repositorio remoto

Reemplaza "master" con la rama en la que quieres enviar los cambios cuando no quieras enviarlos a la rama maestra.

git branch

se usa para listar, crear o borrar ramas.

Para borrar una rama git branch -d "branch-name"

git pull y git merge

git pull fusiona todos los cambios que se han hecho en el repositorio local con el directorio de trabajo local.

git merge se usa para fusionar una rama con otra rama activa
